# docs/founder

For me, when I forget.

Each file here is an FAQ on something I keep re-learning. Not for hires (yet).
Not for users. Not for marketing.

**Format:** questions, not procedures. SOPs assume process; FAQs map to how a
founder actually thinks. Borrowed from Chesky's founder-mode framing — stay in
the details, write for myself, not for an org chart that doesn't exist.

**Rule for adding a new file:** wait until I've felt a question's pain twice.
Three docs that each saved me 30 minutes beat twelve docs that look complete.
