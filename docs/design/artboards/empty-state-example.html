Notes for implementer

1. The state chip, title, desc, primary button, and crystal grid already exist in public/index.html inside <section class="hero-card">. The primary button is wired to App.runHeroAction() — keep that hook.
2. Empty state differs from other states by hiding the chip and hiding the crystal grid. All other states (growing, fractured, hibernating, actualized) keep both visible.
3. New classes introduced: .hero-eyebrow, .hero-secondary-action, .hero-kbd-hint. Everything else reuses existing classes.
4. Copy is the breaking change. Do not ship partial copy updates — the three strings (eyebrow, title, desc) form one Socratic voice unit.
