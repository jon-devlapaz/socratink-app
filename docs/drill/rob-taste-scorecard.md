# Rob Taste Scorecard

Purpose: quick qualitative scorecard for tasting the drill loop as a product experience, not just a correctness system.

Use this after a short fixture run or manual drill session.

Read first:

- [ux-framework.md](../product/ux-framework.md)
- [answer-grid-template.md](answer-grid-template.md)
- [tasting-harness.md](tasting-harness.md)

This scorecard is for emotional and pedagogical feel.
It does not replace backend correctness checks.

## How To Use It

Run one node through:

- explicit unknown
- mixed hesitant attempt
- shallow attempt
- deep attempt
- misconception
- solid
- masterful solid

Then fill this in quickly.
Do not overthink the wording.

## Keep / Change / Watch

### Explicit Unknown

Keep:

- does help feel safe?
- does the learner remain “in the game”?

Change:

- does scaffold drift into lecture mode?
- does the tutor give too much away?

Watch:

- any hint of graded shame
- any subtle performance cue on help-seeking

### Mixed Hesitant Attempt

Keep:

- does the tutor notice the learner is reaching?
- does it preserve momentum?

Change:

- does it get flattened into generic shallow feedback?
- does it sound too teacherly?

Watch:

- does the system miss the mechanistic claim and treat it like help mode?

### Shallow

Keep:

- does the probe target one concrete missing link?

Change:

- does it sound like school feedback instead of live sparring?
- does “good start” become repetitive filler?

Watch:

- shallow and deep collapsing into the same feel

### Deep

Keep:

- does it feel like a near-miss with promise?
- does the learner feel close to breakthrough?

Change:

- does the response feel too similar to shallow?
- is there too little reward for partial structure?

Watch:

- whether “deep” feels emotionally distinct enough to matter

### Misconception

Keep:

- does the tutor redirect without scolding?
- does it offer a cleaner foothold?

Change:

- does the correction feel textbooky?
- does it over-explain instead of repairing the model?

Watch:

- shame tone
- overly long correction

### Solid

Keep:

- does the node resolution feel earned?
- does the state transition feel meaningful?

Change:

- does the payoff feel administratively correct instead of satisfying?

Watch:

- whether an ordinary solid answer feels too similar to masterful solid

### Masterful Solid

Keep:

- does this feel qualitatively different from ordinary solid?
- does it create a real “yes, that landed” moment?

Change:

- if the system treats it as merely another correct answer

Watch:

- rewarding verbosity instead of clarity
- rewarding confidence instead of mechanism reconstruction
