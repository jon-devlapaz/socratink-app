# Cluster-Hosted Rebuild Run Product Chat 2026-04-14

## Summary
This source captures a product discussion about whether Socratink should add a quiz-like feature and, if so, how to distinguish it from Repair Reps. The discussion rejected conventional quiz framing because Generation Before Recognition and truthful graph state require generated reconstruction rather than recognition or aggregate scores.

The emerging product shape is a cluster-hosted, node-resolved **Rebuild Run**. A cluster can act as the learner-facing doorway into an integrated return session, but each scored step remains bound to one eligible child node and resolves through the existing spaced re-drill semantics. The cluster itself does not receive a direct mastery state; its state remains derived from child node outcomes.

The discussion also captured a deliberate deferral. The user likes the feature idea, but does not want to implement it yet and wants to continue measuring runs first. This source should therefore inform future planning without entering the active release queue.
