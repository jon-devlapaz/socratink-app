# Socratinker Product Memory

This knowledge base captures durable product memory for Socratink without creating a second planning system.

Zen of Socratinker:
- Raw before compiled.
- Evidence before inference.
- Source before synthesis.
- Active only when release-relevant.
- Missing instrumentation is evidence.
- Validation before trust.
- Small docs beat duplicated docs.

Its current scope is intentionally narrow:
- external repo audits and critique documents
- product evidence and release findings
- promoted issues, decisions, and syntheses that materially affect the MVP gate

Raw artifacts belong under `raw/`.
Compiled memory belongs under `wiki/`.

Use `ACTIVE.md` only for validated, release-relevant work. Research notes, critiques, and product ideas that do not affect the current MVP gate should stay in compiled memory without becoming active tasks.
